    armageddon¶

    The armageddon command generates Arma configuration material and supports orderer test/load workflows.

    armageddon¶

    usage: armageddon [<flags>] <command> [<args> ...]

Utility for generating Arma config material

Flags:
  --help  Show context-sensitive help (also try --help-long and --help-man).

Commands:
  help [<command>...]
    Show help.

  generate [<flags>]
    Generate config material

  showtemplate
    Show the default configuration template needed to build Arma config material

  version
    Show version information

  submit [<flags>]
    Submit txs to routers and verify the submission

  load [<flags>]
    Submit txs to routers and verify the routers have received the txs

  receive [<flags>]
    Pull txs from some assembler and report statistics

  createSharedConfigProto [<flags>]
    Create a shared config binary from a shared configuration YAML file

  createBlock [<flags>]
    Create a new block with the given shared configuration

    armageddon generate¶

    usage: armageddon generate [<flags>]

Generate config material

Flags:
  --help                  Show context-sensitive help (also try --help-long and
                          --help-man).
  --output="arma-config"  The output directory in which to place config files
  --config=CONFIG         The configuration template to use
  --useTLS                Defines if the connection between a client to a router
                          and an assembler is a TLS one or not
  --sampleConfigPath=SAMPLECONFIGPATH
                          The path to the sample config files
  --clientSignatureVerificationRequired
                          Specify if client signature verification is required

    armageddon showtemplate¶

    usage: armageddon showtemplate

Show the default configuration template needed to build Arma config material

Flags:
  --help  Show context-sensitive help (also try --help-long and --help-man).

    armageddon version¶

    usage: armageddon version

Show version information

Flags:
  --help  Show context-sensitive help (also try --help-long and --help-man).

    armageddon submit¶

    usage: armageddon submit [<flags>]

Submit txs to routers and verify the submission

Flags:
  --help                       Show context-sensitive help (also try --help-long
                               and --help-man).
  --config=CONFIG              The user configuration needed to connection with
                               routers and assemblers
  --transactions=TRANSACTIONS  The number of transactions to be sent
  --rate=RATE                  The rate specify the number of transactions per
                               second to be sent
  --txSize=512                 The required transaction size in bytes

    armageddon load¶

    usage: armageddon load [<flags>]

Submit txs to routers and verify the routers have received the txs

Flags:
  --help                       Show context-sensitive help (also try --help-long
                               and --help-man).
  --config=CONFIG              The user configuration needed to connection with
                               routers
  --transactions=TRANSACTIONS  The number of transactions to be sent
  --rate=RATE                  The rate specifies the number of transactions
                               per second to be sent as one or more rate numbers
                               separated by space
  --txSize=TXSIZE              The required transaction size in bytes

    armageddon receive¶

    usage: armageddon receive [<flags>]

Pull txs from some assembler and report statistics

Flags:
  --help            Show context-sensitive help (also try --help-long and
                    --help-man).
  --config=CONFIG   The user configuration needed to connection with assemblers
  --expectedTxs=-1  The expected number of transactions the assembler should
                    received
  --output="."      The output directory in which to place statistics file
  --pullFromPartyId=PULLFROMPARTYID
                    The party id of the assembler to pull blocks from

    armageddon createSharedConfigProto¶

    usage: armageddon createSharedConfigProto [<flags>]

Create a shared config binary from a shared configuration YAML file

Flags:
  --help           Show context-sensitive help (also try --help-long and
                   --help-man).
  --sharedConfigYaml=SHAREDCONFIGYAML
                   The path to the shared configuration YAML file
  --output=OUTPUT  The output directory in which to place the shared config
                   binary

    armageddon createBlock¶

    usage: armageddon createBlock [<flags>]

Create a new block with the given shared configuration

Flags:
  --help                     Show context-sensitive help (also try --help-long
                             and --help-man).
  --sharedConfigYaml=SHAREDCONFIGYAML
                             The path to the shared configuration YAML file
  --blockOutput=BLOCKOUTPUT  The output directory in which to place the block
  --baseDir=BASEDIR          The directory in which all crypto and config
                             material is saved
  --sampleConfigPath=SAMPLECONFIGPATH
                             The path to the sample config files
